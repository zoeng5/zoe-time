## 1. 睡眠目标"29%"从何而来？

### 源代码位置
📍 `/Users/zoe/AI/Workspaces/moodytiger/timekeeper-feishu-bridge/proactive.py:44`

```python
SLEEP_MIN = 7.0        # 每晚 ≥7h
```

### 目标计算逻辑
- **硬目标**：每晚睡眠 ≥ 7 小时
- **百分比换算**：
  - 一周 = 168 小时
  - 每晚 7 小时 × 7 晚 = 49 小时
  - 百分比 = 49 ÷ 168 = **29.17% ≈ 29%**

所以 **29% 目标在逻辑上是正确的**（基于 7h/晚 的护栏）。

## 4. 为什么目标设定"不合理"？

从代码来看，目标**逻辑上是合理的**：
- 7h/晚 是行业标准（WHO 建议 7–9h）
- 29% 是正确的周百分比

**但用户感受不对的原因可能是：**

1. **个人差异**
   - 用户可能自觉需要 7.5h–8h（不是 7h）
   - 目标应该 **个性化**：改为 7.5h 或 8h？

2. **护栏太宽松**
   - "≥7h 就算达成"，但用户说"整体睡眠不够"
   - 说明用户的实际睡眠低于 7h
   - **系统应该记录实际睡眠值，而不是只用目标 check**

3. **目标与现实脱节**
   - 设定了 7h 目标，但数据管道断了，无法验证是否达成
   - 用户在「盲目」执行一个看不见数据反馈的目标
